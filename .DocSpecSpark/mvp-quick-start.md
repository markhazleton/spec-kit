# MVP Quick Start - DocSpecSpark "Spark" Edition

## What is the MVP?

**DocSpecSpark Phase 1** is a Git-based document management system with these components:

1. **Constitution (Single Source of Truth)** - Company metadata repository (`constitution.yaml`)
2. **CLI tool** - Python command-line tool for creating and publishing documents
3. **Template Token Replacement** - Templates use `{{TOKENS}}` replaced from constitution
4. **Markdown files** - All documents stored as `.md` files in Git
5. **Static site** - Docusaurus website for viewing documents (no login required)
6. **Git versioning** - Semantic version tags (`v1.0`, `v2.0`) for robust version control
7. **GitHub Pages** - Free static site hosting

## Core Architecture: Constitution as Single Source of Truth

**Key Principle**: All company metadata lives in ONE place → the constitution.

```
constitution.yaml (SINGLE SOURCE)          Templates (WITH TOKENS)
├── company.legal_name                  →  {{COMPANY_NAME}}
├── company.executives.ceo.name         →  {{CEO_NAME}}
├── company.headquarters.state          →  {{STATE}}
├── company.department_heads.ehs.name   →  {{EHS_MANAGER}}
└── ... (all metadata)                  →  ... (all tokens)

When you create a document:
  1. CLI loads constitution.yaml
  2. CLI loads template (with {{TOKENS}})
  3. CLI replaces all tokens with constitution values
  4. CLI saves completed document to documents/
```

**Result**: Update CEO name once in constitution → All future documents use new name automatically.

**DocSpecSpark** installs into your company repository just like Spec Kit:

```bash
# Install DocSpecSpark framework into existing repo
$ docspec init

║  (This will create your company metadata repository)    ║
╚══════════════════════════════════════════════════════════╝

Select your organization type:
  1. Not-for-Profit (501(c)(3))
  2. Small Business - Manufacturing
  3. Small Business - Service
  4. Early-Stage Startup
  5. Healthcare Provider
  6. Mid-Sized Enterprise
  7. Large Enterprise
  8. Custom

Your choice [1-8]: 2

# Interactive questionnaire populates constitution...
# (See company-constitution-schema.md for full example)

Legal company name: Acme Manufacturing Inc.
CEO: John Smith
COO: Sarah Johnson
EHS Manager: David Martinez
Primary facility address: 500 Industrial Parkway, Toledo, OH 43612
...

✓ Constitution created: .docspecspark/constitution.yaml
✓ Installed 40 manufacturing templates
✓ Configured Docusaurus site
✓ Added GitHub Actions workflow
✓ Ready to create documents!

# Your repository structure after installation:
my-company-docs/
├── .docspecspark/          # ← ALL framework content lives here
│   ├── constitution.yaml   # ← SINGLE SOURCE OF TRUTH for company metadata
│   ├── scripts/            # Bash + PowerShell scripts
│   ├── templates/          # 61 document templates (with {{TOKENS}})
│   ├── prompts/            # AI agent prompts (future)
│   └── site/               # Docusaurus theme
├── documents/              # ← YOUR company documents (tokens replaced)
│   ├── governance/
│   ├── safety/
│   ├── quality/
│   ├── hr/
│   └── it-security/
├── .github/workflows/
│   └── publish.yml         # Auto-generated by DocSpecSpark
└── README.md
```

**Key Insight**: The `.docspecspark/` folder is like `.documentation/` in Spec Kit:
- Contains ALL framework code (scripts, templates, prompts)
- **Constitution is the ONLY place company metadata lives**
- Can be updated independently (`docspec update` pulls latest framework)
- Never edit manually (update from source repo)
- Company documents live OUTSIDE this folder

**Source vs. Target Repositories**:
- **Source**: `github/docspecspark` - Framework development
- **Target**: `acme-corp/docs` - Company documents (installs framework via CLI)

## Framework Update Pattern (Like Spec Kit)

```bash
# Source repository publishes new version
$ cd docspecspark/
$ git tag v1.2.0
$ git push --tags
# GitHub Release created with .docspecspark.zip artifact

# Target repository pulls update
$ cd acme-corp-docs/
$ docspec update
Checking for updates...
Found DocSpecSpark v1.2.0 (current: v1.1.0)

Updating framework:
  ✓ Downloaded v1.2.0
  ✓ Updated .docspecspark/scripts/
  ✓ Updated .docspecspark/templates/
  ✓ Updated .docspecspark/site-theme/
  ⚠ Skipped documents/ (your content)
  ⚠ Skipped .github/workflows/ (local customizations)

Update complete! Framework version: v1.2.0

Changelog:
- Added 5 new templates (healthcare focus)
- Improved publish script performance
- Fixed Docusaurus theme bug
```

**Safety**: Updates only touch `.docspecspark/` folder, never your `documents/` directory.
